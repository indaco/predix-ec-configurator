Scenario 1 - Connect on-premise data sources to Predix cloud.

Selecting this scenario you are going to configure and use Enterprise Connect to connect on-premise data sources to Predix Cloud.

What Do you need for it?

  • Push a sample application to Predix
  • Create the UAA service instance
  • Bind the sample application to the UAA service instance.
  • Create an instance of the Enterprise Connect service.
  • Create an OAuth2 client to setup access to EC service authenticated using UAA
  • Update the OAuth2 client to add service specific scopes or authorities.
  • Bind the application to the service instance.


Read the documentation on Predix.io to create and configure the Enteprise Connect service.


Note:EC-Configurator requires a sample app binded to the predix services in order to retrieve information from VCAPS and automate the configuration.

Scenario 2 - Connect Predix data sources to on-premise systems

Selecting this scenario you are going to configure and use Enterprise Connect to connect Predix data sources to on-premise systems.

What Do you need for it?

  • Push a sample application to Predix
  • Create the UAA service instance
  • Bind the sample application to the UAA service instance.
  • Create an instance of the Enterprise Connect service.
  • Create an OAuth2 client to setup access to EC service authenticated using UAA
  • Update the OAuth2 client to add service specific scopes or authorities.
  • Bind the sample application to the service instance.
  • Create the CF Datasource service instance (PostgreSQL and/or RabbitMQ)
  • Bind the sample application to the service instance.


Read the documentation on Predix.io to create and configure the Enteprise Connect service.


Note:EC-Configurator requires a sample app binded to the predix services in order to retrieve information from VCAPS and automate the configuration.
